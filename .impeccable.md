## Design Context

### Users
Developpeurs francophones, debutants et experimentes, qui utilisent ou decouvrent Claude Code. Ils consultent le site comme reference rapide pendant leur travail ou pour apprendre les fonctionnalites (skills, hooks, commandes, MCP, settings).

### Brand Personality
Accessible, pedagogique, utilitaire. Comme un guide bien fait par un ami dev — clair, direct, pratique.

### Aesthetic Direction
- **Tone**: Light mode, chaleureux mais professionnel. Inspire par awesomeclaude.ai mais en version claire.
- **Reference principale**: awesomeclaude.ai — grid layout, cards avec hover effects, navigation par categories, code blocks soignes.
- **Palette**: Fond sable clair (#faf8f5), accents orange Claude (#e07c3e), texte brun (#4a3728), cards blanches avec bordures sable, hover orange.
- **Typo**: Display font distinctive pour les titres + body font lisible. Pas Inter/Roboto (trop generique).
- **Anti-references**: Pas de dark mode, pas de gradients neon, pas de glassmorphism excessif.
- **Langue**: Tout le contenu en francais.

### Design Principles
1. **Scannable** — L'information doit se trouver en 3 secondes. Hierarchie visuelle claire, tables, grids de cards.
2. **Copy-paste ready** — Les blocs de code sont le coeur du site. Ils doivent etre beaux, lisibles, avec possibilite de copier facilement.
3. **Chaleureux, pas froid** — Tons sable/terre, pas de gris clinique. Le site doit donner envie de rester.
4. **Multi-pages navigable** — Navigation claire entre sections, chaque page auto-suffisante.
5. **Progressive** — Debutants trouvent les bases, experimentes trouvent les details avances.
